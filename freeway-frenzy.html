FreewayFrenzy

FreewayFrenzy runs across two microcontrollers, splitting input and rendering responsibilities over UART. The page now embeds a browser simulation of that loop so the actual pacing, lane logic, HUD, and traffic difficulty are visible instead of being described abstractly.

Embedded here: a game-surface port of the project’s lane-switching and distributed-state loop.

Built With

ESP32-S3 ESP32-C3 C ESP-IDF LVGL UART

System Split

One controller handles player input while the display-side controller owns rendering and game-state presentation. That split is part of the point of the project, not an implementation detail.

Game Logic

  • Lane switching is discrete and optimized for low-latency input hardware
  • Traffic cadence ramps with score to validate timing on-device
  • HUD and render loop communicate state clearly under tight constraints

Embedded Focus

This was not just a small game. It was a testbed for communication between constrained devices, deterministic rendering, and responsive control surfaces on a microcontroller-class platform.

Why Simulate It Here

A static page cannot communicate the feel of the loop. Embedding a browser port of the actual mechanics makes the architecture legible because users can see how the state machine behaves under input.
