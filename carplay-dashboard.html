What The Embed Shows

A browser-runtime stand-in for the hardware interface, demonstrating the animation pacing, state transitions, and compositional density that define the real embedded dashboard.

Key Product Logic

  • Speed and telemetry update as one coherent vehicle state
  • Navigation and media compete for attention without clutter
  • All major information blocks fit a fixed automotive viewport

Embedded Constraints

Designing for a 1024×600 automotive display forces hard prioritization. Touch targets, typography, animation curves, and panel groupings need to read at a glance instead of feeling like a resized tablet app.

Portfolio Fit

A browser simulator is the right bridge here because the original target hardware is not web-native. It still demonstrates the actual design logic of the system instead of reducing the project to screenshots and prose.
