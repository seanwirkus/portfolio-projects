Embedded Vision App

RetroVision

RetroVision is a retrofit perception HUD for older vehicles. A camera classifies cars, stop signs, traffic signals, and pedestrians while a TFmini-S LiDAR stream tracks forward distance. The data is fused on an ESP32-C3, sent over UART, and rendered as a real-time ESP32-S3 driver-assist display.

Main experience: the live RetroVision Lab below, with the HUD, scene editor, planner state, and UART-style telemetry kept together.

Built With

ESP32-S3 ESP32-C3 C ESP-IDF LVGL UART TFmini-S

System Split

The ESP32-C3 owns sensor fusion and packet construction. The ESP32-S3 owns the display loop, HUD rendering, and driver-facing alerts.

Data Kept

  • LiDAR distance and range state
  • Detected object class, position, and confidence
  • Planner decision, reason, and telemetry packet preview

Why It Is Separate

RetroVision is no longer presented as a follow-on section inside the driving game project. It is its own embedded vision app with a different purpose: retrofit driver awareness and sensor-fusion validation.

Why The Lab Matters

The lab keeps the useful data visible while making the core product easy to understand. You can test object placement, distance changes, planner behavior, and UART output without opening the older driving game.
